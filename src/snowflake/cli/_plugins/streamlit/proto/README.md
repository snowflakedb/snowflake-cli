## Streamlit developer log streaming protobuf definitions

This directory contains the protobuf definitions used by the
`snow streamlit logs` command to talk to the Streamlit container runtime's
developer log service.

### Files

- `developer/v1/logs_service.proto` — source proto definition (authoritative).
- `generated/developer/v1/logs_service_pb2.py` — generated Python bindings.
  Do not edit by hand.

### When to regenerate

Regenerate `logs_service_pb2.py` any time `logs_service.proto` is modified
(new fields, enum values, messages, etc.) or when bumping the protobuf
toolchain. The generated file must be committed alongside the proto change
in the same PR.

### How to regenerate

The project pins `protobuf>=5.29.6,<6` (see `pyproject.toml`). You must
regenerate using a protobuf 5.x toolchain so the
`_runtime_version.ValidateProtobufRuntimeVersion(...)` call emitted at the top
of the generated file targets the supported runtime range. Generating with
protobuf 6.x would require a fragile `try/except` shim and could mask real
version mismatches at import time.

From the repository root:

```bash
python -m venv /tmp/pbgen
/tmp/pbgen/bin/pip install 'protobuf==5.29.6' 'grpcio-tools<1.69'

/tmp/pbgen/bin/python -m grpc_tools.protoc \
  --proto_path=src/snowflake/cli/_plugins/streamlit/proto \
  --python_out=src/snowflake/cli/_plugins/streamlit/proto/generated \
  developer/v1/logs_service.proto
```

### Post-generation manual edits

`protoc` emits a bare `# Generated by ...` header. After regenerating,
re-apply the small header block in `logs_service_pb2.py`:

- The regeneration-command comment (matches this README so the file is
  self-documenting).
- The `# ruff: noqa: SLF001` directive (the generated body uses `_globals[...]`
  private accessors that ruff would otherwise flag).
- Move the `from google.protobuf import timestamp_pb2 as _timestamp_pb2`
  import into the top import block and tag it
  `# noqa: F401  # side-effect: registers Timestamp in descriptor pool` —
  protoc emits this import below the `_sym_db` assignment with an
  unconventional alias and no marker, so ruff strips it as unused. The
  import is load-bearing because `LogEntry.timestamp` references the
  `google.protobuf.Timestamp` descriptor, which must already be in the
  pool before `AddSerializedFile` runs.

### Source of truth

The proto definition is mirrored from
`github.com/snowflakedb/streamlit-container-runtime/gen/developer/v1` — if
the server-side definition changes, update `logs_service.proto` here to
match before regenerating.
